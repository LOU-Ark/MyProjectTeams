# WBS Designer Skill

## Goal
要求定義・要件定義を、フェーズベースの WBS へ変換する。

## Instructions
1. 要件を以下の観点で整理する:
   - フェーズ
   - 成果物
   - 実装対象
   - テスト対象
   - レポート対象
2. WBS は 2〜3 階層程度で読みやすく保つ。
3. 各タスクがどの要件に対応しているか追跡できるようにする。
4. `03_planning/wbs.md` を生成する。

## Recommended Structure
- 1. 要件確認
- 2. 情報設計
- 3. フォーム入力機能
- 4. ファイル読込機能
- 5. チャット補完機能
- 6. ブリーフ生成機能
- 7. 品質確認
- 8. リリース準備

## Output Targets
- `03_planning/wbs.md`

## Constraints
- 要件に紐づかないタスクを作らない。
- テストとレポートを WBS から省略しない。
