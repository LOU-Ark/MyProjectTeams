# MVP Refinement Workflow

## Step 1: Review MVP
- 現在のMVPの成果物、できている点、足りない点を整理する。

## Step 2: Collect Improvement Requests
- requirements-gathering を使って改善要求を収集する。
- 変更理由、期待効果、緊急度、影響範囲を整理する。

## Step 3: Prioritize
- 改善要求を Must / Should / Could / Won’t に分類する。
- すぐやるものと後回しにするものを分ける。

## Step 4: Create Next WBS
- pmbok-project-management を使って改善WBSを作る。
- 1〜2週間単位で実行可能な粒度にする。

## Step 5: Confirm Next Iteration
- 次回の改善スコープをユーザーと確認する。
