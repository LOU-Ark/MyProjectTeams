# RULE: 09-mermaid-standardization

## 概要
プロジェクト内でのすべての図解（Mermaid）は、標準テンプレートに基づき、一貫したスタイルで生成されなければならない。

## 強制遵守事項
1. **テンプレートの参照**:
   Mermaid図を作成・更新する際は必ず `.agent/templates/99-mermaid-templates.md` を参照し、定義されている共通テーマ設定（init）を冒頭に含めること。

2. **配色の統一**:
   プロジェクトのブランドカラー（Primary: #4f46e5, Secondary: #06b6d4）を使用し、視認性の高いダイアグラムを作成すること。

3. **日本語対応とラッピング**:
   図中のテキストは日本語とし、要素内のテキストが1行あたり「全角10文字」を超える場合は、必ず `<br/>` を用いて明示的に改行を行うこと。また、特殊文字による不具合を防ぐため、ラベル全体を二重引用符（"）で囲むことを必須とする。

4. **アイコンの活用**:
   必要に応じて Unicode 文字やアイコンラベルを使用し、直感的に理解しやすい図解を心がけること。

## 適用場面
- インシデントレポート（原因分析フロー）
- フェーズ完了報告（進捗・状態遷移）
- システムアーキテクチャの説明
