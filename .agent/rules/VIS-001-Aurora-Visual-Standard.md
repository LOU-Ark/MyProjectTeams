# RULE: VIS-001 - Aurora Visual Identity & Structural Design

> [🏠 00_Master ] > [ .agent ] > [ rules ] > VIS-001-Aurora-Visual-Standard

## 1. 構造の美学 (3-Tier Architecture)
美しさはファイル構造から始まる。規約 `Ag-Mng-Prd` を厳守し、論理的な整理を徹底せよ。
- **.agent/**: AI の知能、スキル、規約。
- **management/**: ビジネスロジック、要件、履歴、ポータル。
- **product/**: 最終製品ソースコード。
- **Principle**: ルートディレクトリのクリーンさを保つこと。

## 2. 視覚的アイデンティティ (UI/UX Standards)
製品の UI は、以下の「Aurora Visual Standard」に準拠しなければならない。
- **Layout**: 40px 以上の外周余白、コンポーネント間の適切な余白。
- **Rounding**: カード・ボタン等の要素には 20px〜30px の角丸を適用。
- **Backdrop**: グラスモーフィズム（透過・ぼかし）と、Aurora グラデーションの融合。
- **Typography**: Google Fonts "Outfit" または "Inter" の使用。

## 3. インタラクション (Micro-interactions)
- **Smoothness**: メッセージの登場やボタンのホバーには、0.3s 程度のイージングアニメーションを付与せよ。
- **Visual Feedback**: アクションに対し、即座に視覚的な応答（ハイライト・影の変化等）を返すこと。

---
**Vision**: 透明感、滑らかさ、先進性の融合。
**Policy**: デザインの美しさは「妥協なき品質」の一部である。
