# 🧹 Resource Hygiene Standard (リソース清潔規約)

## 1. ブラウザ・タブの管理
- **原則**: 検証（browser_subagent）が終了した際、または特定のページが不要になった際は、即座に **`about:blank`** へ遷移させてメモリリソースを解放しなければならない。これは「後回し」にせず、検証の直後に行う。
- **指示の徹底**: ブラウザの閉じ忘れは、チームの「リソース管理能力不足」と見なされる。セッションの最後には必ず `list_browser_pages` で残りがないか点検すること。

## 2. バックグラウンドプロセスの終了
- **原則**: `npm run dev` 等のサーバープロセスは、検証が完了した時点で速やかに **停止（Terminate）** させなければならない。
- **指示の優先**: ユーザーから「サーバーを動かしたままにして」という明示的な指示がない限り、作業終了時のプロセス生存は「不備」とみなす。

## 3. 目的
- リソース競合による誤動作の防止。
- ユーザー環境のパフォーマンス維持。
- プロフェッショナルな作業品質の証跡（跡を濁さない）。
