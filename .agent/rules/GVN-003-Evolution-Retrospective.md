# 📊 Project Evolution Tracking Standard (進化追跡規約)

## 1. 原則
全てのプロジェクトおよび主要なイテレーションの完了時に、チーム自身の属性（Awareness, Quality, Governance）がどのように変容し、成長したかを「Before/After」で明文化し、プロジェクトアーカイブに残さなければならない。

## 2. 必須比較項目
1. **初期状態（その時点でのベスト）**: 最初に出力した際の課題点や、未成熟だったプロセス。
2. **フィードバックの受容**: ユーザー様、または内部検証で見つかった指摘事項。
3. **現在の状態（進化後）**: 上記指摘を受けて改善された後の、チームの「新・規律」。

## 3. レポーティングの責務
`07_Team_Analytics/` フォルダ配下に **`Evolution_Report`** を作成し、それをポータルの「チーム分析」カテゴリに掲載すること。

## 4. 目的
- 単発のプロジェクト完遂で終わらず、チームの「知の螺旋形発展」を記録する。
- 変化に対する柔軟性と学習能力の証跡（ガバナンス）を提示する。
- プロフェッショナルとしての「振り返りと成長」の仕組み化。
