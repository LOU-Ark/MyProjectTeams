# 🔄 Issue Feedback Loop Standard (課題改善フィードバック規約)

## 1. 原則
チームがインシデント（INC）を検出し、`05_Issue_Log` に記録した際、その解決策を「一時的な修正」に留めてはならない。必ず**再発防止策を「チームのルール」または「改善計画書」に即時反映**させなければならない。

## 2. 実行手順
1. **INC記録**: 事象・原因・対策を `05_Issue_Log` に記述。
2. **ルール更新**: 対策に基づき、`.agent/rules/` 内の該当規約を書き換えるか、新規規約を制定する。
3. **レポーティング**: `07_Team_Analytics/` にて、その不備がどのようにチーム能力の向上（仕組み化）に繋がったかをレポートする。

## 3. 目的
- 同じミスを二度繰り返さない「自ら学ぶチーム」の実現。
- プロの手仕事としてのガバナンス（統治）の証跡。
- プロジェクトの継続的な品質改善。

## 4. 遵守確認
イテレーションの最後において、新規 INC が発生しているにもかかわらず規約（rules）の更新がない場合、その納品準備は「不適格」とみなす。
