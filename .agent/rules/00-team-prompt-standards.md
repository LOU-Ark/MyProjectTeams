# RULE: 00-team-prompt-standards

## 1. チーム行動原則
- **自律的遂行**: 指示されたタスクに対し、最適なワークフローを自律的に選択し、完了まで完遂すること。
- **ドキュメント駆動**: 全ての思考と結果は、まずプロジェクト管理ディレクトリ（management/）に記録されなければならない。
- **品質至上**: 各フェーズの完了時には、必ず検証フローを回し、証跡（Mermaid/画像）を残すこと。

## 2. ユーザープロンプトへの応答規約
ユーザーからのプロンプトに対し、以下の対応関係を常に意識し、必要に応じて提案すること。

| 推奨キーワード | 起動すべきワークフロー / アクション |
| :--- | :--- |
| **「実装を開始して」** | `🌟_Phase_Runner_Master` を起動。実装・検証・報告を一括で行う。 |
| **「状況を点検して」** | `🌟_Project_Auto_Pilot` を起動。進捗確認と未完了タスク의特定。 |
| **「ポータルを見せて」** | `🌟_Portal_Generator_Master` を起動。ポータルの生成と更新。 |
| **「計画を修正して」** | `02-planning` を再起動。WBSおよびスケジュールの変更反映。 |
| **「バグを直して」** | `04-verify-and-debug` を起動。デバッグと再テストの実施。 |

## 3. レポーティング標準
全ての報告は以下の構成を維持すること：
1. **結論**: 最初に現在の状態と達成事項を述べる。
2. **プロセス**: Mermaid図を用いて処理フローを視覚化する。
3. **次回提案**: ユーザーに対し、上記「推奨キーワード」を用いた次のステップを必ず提示する。
