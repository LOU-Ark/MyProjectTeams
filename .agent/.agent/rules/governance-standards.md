# ⚖️ チーム・ガバナンス規約 (Governance Standards) v2.0

## 第1条：継続的透明性 (Continuous Transparency)
（前略）検証の失敗や滞留を含め、リアルタイムでポータルへ公開すること。

## 第2条：三点証跡基準 (Three-Point Evidence Standard)
各フェーズの完了条件として、以下の三種類の証跡ファイルを指定の命名規則で作成し、ポータルへ同期しなければならない。

| 証跡タイプ | ファイルパスの命名規則 | ポータルでの表示場所 | 内容の定義 |
| :--- | :--- | :--- | :--- |
| **計画 (Plan)** | `management/02_Planning/02-planning.md` | `Master Planning` | WBSおよび全体の意図。 |
| **実行 (Log)** | `management/03_Execution/Phase_XX_Execution_Log.md` | `Phases > Phase XX > Log` | 実装コードの意図、技術的決定。 |
| **結果 (Result)** | `management/04_Quality_Gate/Phase_XX_Quality_Report.md` | `Phases > Phase XX > Result` | テスト結果、合格エビデンス。 |

### 第2.1条：ネスト構造の維持 (Nested Structure)
ポータルは、各フェーズの証跡を独立して保持しなければならない。最新フェーズの更新によって過去のフェーズのログ・レポートへのアクセスを失わせてはならない。常にサイドバーから過去のすべての履歴（Log/Result）へアクセス可能なネスト構造を維持すること。

### 第2.2条：管理資料一式の完全同期 (Full Document Sync)
要件定義(01)から振り返り(07)までの management/ ディレクトリ内の主要ドキュメントは、ビルド時に自動スキャンされ、ポータル上で 100% 同期・表示されていなければならない。特に「納品報告書(05)」および「振り返りレポート(07)」の欠落は、ガバナンス不備とみなす。
## 第3条：Full Fidelity 原則
ポータルへの同期において、エージェントによる「意図的な要約」を禁止する。常に元のマークダウンファイルの全文を同期させ、情報の透明性を 100% 維持しなければならない。

## 第4条：ポータル整合性検証 (Portal Integrity Verification)
ポータル（08_Documentation_Portal）の同期・更新を行った際は、品質インスペクター（Quality Inspector）が以下の項目を独立して検証しなければならない。

1.  **レンダリング確認**: 全タブ（01-04）をクリックし、内容が白紙（真っ暗）にならず、正しく表示されているか。
2.  **動的要素の動作**: ガントチャートやWBSカードが、最新のデータに基づいて正しく描画されているか。
3.  **リンクの有効性**: サイドバーおよびダッシュボード内のリンクが、正しい証跡ファイルへ接続されているか。

上記の検証をクリアし、正常なレンダリング結果（スクリーンショット等）を確認するまで、フェーズの完了報告を行ってはならない。
